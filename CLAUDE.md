# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Custom Planck 4x12 ortholinear mechanical keyboard PCB. MCU: Raspberry Pi Pico 2 (RP2350). Features: Choc v1/v2 hotswap switches, per-key SK6812mini-e RGB LEDs, bare 1.3" SSD1306 OLED panel (FPC), EC11 rotary encoder.

PCB is generated with **ergogen** (folder mode) using ceoloide community footprints, then manually edited in **KiCad 9.0**.

## Commands

```bash
# Regenerate PCB from ergogen config (WARNING: overwrites manual KiCad edits)
npx ergogen . -o output --clean

# Install dependencies
npm install
```

Open `output/pcbs/p48v2.kicad_pcb` in KiCad pcbnew for manual editing. The KiCad MCP server can also be used for programmatic board manipulation.

## Architecture

### Pipeline

```
config.yaml (ergogen) → npx ergogen → output/pcbs/p48v2.kicad_pcb → KiCad manual edits
```

The ergogen config is the **source of truth** for switch placement, LED chain, diode positions, and board outline. Post-generation edits (component placement, net assignment, zones, routing) are done in KiCad and are **not tracked by ergogen** — regenerating will destroy them.

### Key Files

- `config.yaml` — Ergogen config: 4x12 matrix definition, column/row nets, LED chain (serpentine), outline (255x72mm board = 220mm switches + 35mm extension), footprint params
- `output/pcbs/p48v2.kicad_pcb` — Main PCB file with all manual edits (component placement, nets, zones)
- `output/pcbs/p48v2.kicad_pro` — KiCad project settings and design rules
- `footprints/ceoloide/` — Custom ergogen footprints (JS): switch_choc_v1_v2, led_sk6812mini-e, diode_tht_sod123
- `thoughts/shared/handoffs/` — Session handoff files (YAML) documenting previous work

### Matrix Design

4x12 key matrix using diodes (16 GPIO: 12 columns + 4 rows) instead of direct-pin (which would need 48 GPIO). Column nets COL0-COL11, row nets ROW0-ROW3. Each key position has a unique `colrow` net (e.g. CR_0_0) connecting switch to diode.

### Component Placement

| Ref | Component | Layer | Notes |
|-----|-----------|-------|-------|
| U1 | Pico 2 THT | B.Cu | Rotated 180°, through-hole for mechanical strength |
| J1 | FPC ZIF 30pin (Hirose FH12-30S-0.5SH) | F.Cu | For bare 1.3" SSD1306 OLED panel |
| C1-C6 | OLED passives (caps) | F.Cu | Charge pump, bypass, decoupling |
| R1-R3 | OLED passives (resistors) | F.Cu | IREF 910k, I2C pull-ups 4.7k |
| RE1 | EC11 Encoder | B.Cu | Rotated -90° |
| - | 48x switches/LEDs/diodes | per config | Generated by ergogen |

### LED Chain

Serpentine pattern across 48 LEDs. Data from GPIO16 → LED0 (top-left) → LED11 (top-right) → LED12 (home-right, reverses) → LED23 (home-left) → LED24 (bottom-left) → LED35 (bottom-right) → LED36 (mod-right, reverses) → LED47 (mod-left). VCC powered directly from VBUS 5V.

### Pin Mapping (Pico)

COL0-COL11: GPIO0-11 | ROW0-ROW3: GPIO12-15 | LED_DATA: GPIO16 | ENC_A/B/SW: GPIO17-19 | I2C_SDA/SCL: GPIO20-21

## KiCad MCP Limitations

These MCP tools are known broken and should not be used:
- `add_zone` — fails silently
- `add_net_class` — fails silently
- `get_board_2d_view` — swig backend error
- `place_component` — breaks after delete+re-place cycles (swig FootprintLoad error); also can't find footprints from standard KiCad libraries
- `find_component` on a deleted reference — freezes the MCP server completely (requires restart)
- `add_net` — creates nets in memory but doesn't persist to PCB file on save
- `export_svg` — reports success but doesn't produce output file

Workarounds: edit the `.kicad_pcb` file directly (S-expression format) or use Python scripts for bulk operations like net assignment and component placement.

## Commit Rules

- **Never add Co-Authored-By** lines to commit messages

## Important Warnings

- **Never regenerate ergogen output** without understanding that all manual KiCad edits (net assignments, component placement for U1/U2/RE1, zones, routing) will be lost
- **Never manually edit footprints** inside the PCB file — this corrupts layer assignments
- The board has GND copper zones on both F.Cu and B.Cu — these need filling in pcbnew (press B) after any edit
- Design rules: default trace 0.25mm / clearance 0.2mm, power trace 0.5mm / clearance 0.3mm
